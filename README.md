# 📐 Пробные ОГЭ/ЕГЭ — платформа для учителей математики

Веб-приложение для проведения пробных экзаменов по математике в школе.
Учитель создаёт варианты и ведёт учеников — ученики решают онлайн, результаты сохраняются автоматически.

---

## Возможности

### Для учителя (админ-панель)
- Создание вариантов вручную, через **каталог заданий** или импортом с **СдамГИА / ФИПИ / PDF**
- Управление классами и учениками (в т.ч. массовый импорт из Excel)
- Просмотр результатов по каждому ученику и классу с аналитикой по номерам заданий
- Ручная проверка заданий части 2 (развёрнутые ответы), в т.ч. массовая проверка сразу нескольких попыток
- Экспорт результатов в `.xlsx`, экспорт аналитики класса в `.xlsx`
- Скачивание вариантов в Word (с ответами / без ответов) или ZIP-архивом
- Уведомления о попытках, ожидающих ручной проверки

### Для ученика
- Вход по ФИО и паролю
- Решение вариантов своего типа (ОГЭ / ЕГЭ профиль / ЕГЭ база) с таймером
- Автосохранение ответов — при закрытии вкладки прогресс не теряется
- Просмотр результатов: правильные/неправильные ответы, время, оценка
- **«Повторить ошибки»** — отдельный вариант только из заданий с неверным ответом
- Личная статистика, история попыток и аналитика по слабым заданиям

## Быстрый старт

### 1. Клонирование и окружение

```bash
git clone <repo-url>
cd EGE
python -m venv env
source env/bin/activate        # Windows: env\Scripts\activate
pip install -r requirements.txt
```

### 2. Настройка

```bash
# Применить миграции
python manage.py migrate

# Создать суперпользователя (учитель-админ)
python manage.py createsuperuser
```

### 3. Запуск

```bash
python manage.py runserver
```

Откройте [http://localhost:8000](http://localhost:8000) — страница входа для учеников.
Админ-панель: [http://localhost:8000/admin/](http://localhost:8000/admin/)

## Каталог заданий

Центральная база заданий, не привязанных к конкретному варианту.

- **Добавление вручную** — через форму
- **Импорт с СдамГИА** — по ссылке на задание или целый вариант (фоново)
- **Импорт из банка ФИПИ** — вставить URL проекта, система покажет кол-во заданий и темы; импорт запускается фоново с live-прогрессом
- **Импорт из PDF («Распечатай и реши»)** — загрузка файлов через кнопку в каталоге; парсинг запускается фоново
- **Защита от дублей** — двойная проверка: по GUID задания (ФИПИ) и по MD5-хэшу текста
- **История импортов** — список всех сессий с датой, кол-вом заданий, статусом; удаление через модальное окно (запись / запись + задания)
- **Неопределённые задания** — вкладка «Без номера»: назначить тип одним кликом; вкладка «Без ответа»: заполнить правильный ответ; фильтр по источнику
- **Создание варианта из каталога** — модальное окно с поиском, фильтрами по источнику, защитой от дублей и индикатором заполненных слотов

### Ручное добавление задания (HTML-форматирование)

Поле **Текст задания** принимает HTML. В форме есть панель кнопок и живой предпросмотр («Как видит ученик»).

| Что нужно | HTML | Результат |
|---|---|---|
| Степень / верхний индекс | `x<sup>2</sup>` | x² |
| Нижний индекс | `a<sub>1</sub>` | a₁ |
| Дробь | `<span class="math-frac"><span>a+1</span><span>b-2</span></span>` | (a+1)/(b-2) вертикально |
| Квадратный корень | `√(x+1)` | √(x+1) |
| Жирный | `<b>текст</b>` | **текст** |
| Курсив | `<i>текст</i>` | *текст* |
| Перенос строки | `<br>` | — |
| Абзац | `<p>текст</p>` | — |

**Сложные формулы:**

```html
<!-- Дробь со степенью в числителе -->
<span class="math-frac"><span>x<sup>2</sup>+1</span><span>2x-3</span></span>

<!-- Корень из дроби -->
√(<span class="math-frac"><span>a</span><span>b</span></span>)
```

**Правильный ответ** — несколько вариантов через `|`:
```
234|243|324
```
Любой из вариантов засчитается как верный. Пробелы, запятые/точки в десятичных и регистр игнорируются при проверке.

### Импорт PDF («Распечатай и реши»)

PDF-сборники содержат блоки вида **«Задание 1.», «Задание 2.»** — каждый блок является отдельным вариантом (не номером ОГЭ-задания). Номера реальных заданий ОГЭ читаются из имени файла (например, `№01-05` → задания 1–5).

Импорт запускается через кнопку **⬇ PDF** в каталоге. Выберите файлы, тип экзамена, формат и куда сохранять — парсинг выполняется в фоне, статус обновляется в реальном времени.

Варианты создаются с `is_active=False`. После проверки ответов активируй их в `/admin/variants/`.

## Импорт учеников из Excel

Перейдите в **Ученики → Импорт Excel**.
Файл должен содержать столбцы: `ФИО`, `Пароль`, `Класс`.
Класс должен уже существовать в системе.

| Переменная            | По умолчанию          | Описание                        |
|-----------------------|-----------------------|---------------------------------|

| `CLOUDINARY_URL`      | —                     | Cloudinary для хранения медиа   |

## Проверка ответов

Ответы нормализуются перед сравнением:

- `0,5` = `0.5` = `1/2`
- Пробелы и регистр игнорируются
- Последовательности: `1;2;3` — порядок важен
- Задания части 2 проверяются учителем вручную

## Деплой

Проект готов к деплою на любой Linux-сервер с Python.

```bash
# Продакшн-сервер
gunicorn config.wsgi:application --bind 0.0.0.0:8000

# Статические файлы раздаёт WhiteNoise — доп. настройка nginx не требуется
python manage.py collectstatic --noinput
```

Для базы данных задайте `DATABASE_URL` в переменных окружения (поддерживается PostgreSQL через `dj-database-url`).
