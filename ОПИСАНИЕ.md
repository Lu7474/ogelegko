# Платформа для пробных ОГЭ/ЕГЭ по математике

Веб-приложение для учителя математики: создавать варианты, назначать классам, собирать результаты.
Ученики решают онлайн с таймером — учитель видит всё в реальном времени.

---

## Для кого

**Учитель** управляет через браузер без установки ПО:
- заводит классы и учеников (или импортирует из Excel)
- создаёт варианты вручную / импортирует из СдамГИА, ФИПИ, PDF
- проверяет часть 2 и видит аналитику по каждому заданию

**Ученик** входит по ФИО и паролю:
- решает вариант своего типа (ОГЭ / ЕГЭ профиль / ЕГЭ база) с таймером
- видит результаты сразу после сдачи
- может «повторить ошибки» — система создаёт вариант только из неверных заданий

---

## Типы экзаменов

| Тип           | Заданий | Время    |
|---------------|:-------:|----------|
| ОГЭ           | 25      | 3ч 55мин |
| ЕГЭ профиль   | 18      | 3ч 55мин |
| ЕГЭ база      | 21      | 3ч       |

Оценки и тестовые баллы считаются по актуальным таблицам ФИПИ (2025/2026).

## Архитектура

```
Django 5.2 (Python 3.11)
├── SQLite (dev) / PostgreSQL (prod)
├── Cloudinary (медиафайлы в проде)
├── WhiteNoise (статика)
└── BeautifulSoup4, PyMuPDF, pdfplumber (парсинг)
```

Приложение одно — `exam`. Представления разбиты по зонам:

| Модуль                    | Зона                                        |
|---------------------------|---------------------------------------------|
| `views/student.py`        | Вход, решение варианта, результаты, профиль |
| `views/admin_base.py`     | Авторизация учителя, дашборд, экспорт       |
| `views/admin_students.py` | Классы, ученики, попытки, аналитика         |
| `views/admin_variants.py` | Варианты, DOCX-печать, ZIP-архив            |
| `views/admin_catalog.py`  | Каталог заданий, импорт (СдамГИА/ФИПИ/PDF) |
| `utils.py`                | Нормализация ответов, таблицы баллов        |
| `parsers/sdamgia.py`      | Парсер СдамГИА, санитизация HTML            |
| `parsers/fipi.py`         | Парсер банка ФИПИ                           |
| `parsers/pdf.py`          | Парсер PDF-файлов                           |
| `services/variant_archive.py` | ZIP-экспорт/импорт вариантов            |
| `services/docx_builder.py`   | Рендеринг вариантов и результатов в DOCX |

## Ключевые модели

- **SchoolClass** — класс с типом экзамена
- **Student** — ученик, вход по ФИО+пароль (хэш bcrypt)
- **Variant** — вариант с лимитом попыток
- **Task** — задание варианта (текст, изображение, правильный ответ, баллы)
- **CatalogTask** — задание в глобальном каталоге (независимо от варианта)
- **Attempt** — попытка ученика (время, балл, оценка)
- **Answer** — ответ на конкретное задание в рамках попытки

---

## Проверка ответов

Ответы нормализуются перед сравнением: `0,5` = `0.5` = `1/2`, пробелы и регистр игнорируются, поддержка π. Правильный ответ может содержать несколько вариантов через `|`.

Задания части 2 (развёрнутые ответы) помечаются флагом `manual_grading` — учитель проверяет вручную через интерфейс массовой проверки.

## Безопасность

- Авторизация учителя — стандартный Django `User` с `is_staff`
- Авторизация ученика — собственная схема (session + session_key в модели)
- IP-based rate limiting на вход (5 попыток / 5 минут)
- Все HTML из внешних источников проходят `sanitize_html()` перед сохранением
- `DJANGO_SECRET_KEY` и `ALLOWED_HOSTS` обязательны в проде (сервер не стартует без них)
- Аудит-лог деструктивных действий учителя в `logs/app.log`
