# Журнал изменений

Все значимые изменения проекта ogelegko.ru фиксируются здесь.

---

## [Текущий] — апрель 2026

### Безопасность и стабильность
- Аудит-лог удаления вариантов, классов, учеников и попыток в `app.log`
- Блокировка удаления заданий варианта, если по ним есть завершённые попытки
- `DJANGO_SECRET_KEY` и `ALLOWED_HOSTS` обязательны в продакшене — сервер не стартует без них
- Исправлена сломанная блокировка `select_for_update` при создании попытки
- Ограничение на загрузку PDF: файлы >50 МБ отклоняются до записи на диск
- Пагинация (50 записей) на странице списка учеников
- Добавлены тесты безопасности: изоляция данных, санитизация XSS, лимит попыток

### Варианты
- **Архив ZIP** — экспорт/импорт вариантов в переносимый ZIP-архив: JSON-метаданные + изображения; атомарный импорт, защита от path traversal, разрешение конфликтов номеров
- Экспорт варианта в Word (версия для учителя с ответами и для ученика)
- Экспорт результатов в таблицу Word (1/0 по каждому заданию)
- SVG-формулы вставляются в DOCX через конвертацию SVG→PNG (PyMuPDF)
- HTML-таблицы из текста задания вставляются в DOCX как таблицы Word
- Таблица ответов в DOCX вынесена на отдельную страницу; более читаемый формат (3 строки: №/Ответ/Балл, серый фон заголовков)
- Несколько дополнительных изображений к заданию (модели TaskImage/CatalogTaskImage)
- Режим «Повторить ошибки» — дополнительные изображения теперь копируются в новый вариант

### Импорт
- Импорт PDF «Распечатай и реши»: режимы `catalog`, `variants`, `both`
- Создание объектов Variant из блоков PDF (каждый блок «Задание N.» = отдельный вариант)
- Импорт из банка ФИПИ (с обходом SSL российского УЦ Минцифры)
- История импортов для всех источников (СдамГИА, PDF, ФИПИ)
- Кнопка «PDF-импорт» добавлена на главную страницу каталога
- Фильтр по источнику на странице «Неопределённые задания»
- Замена dropdown удаления на модальное окно с двумя чёткими кнопками в истории импортов

### Каталог
- Поле `shared_context` и `shared_context_image` для заданий — общее условие для группы заданий
- Два режима страницы «Неопределённые»: задания без номера и без ответа
- Массовое удаление заданий с чекбоксами
- Разбивка `admin_views.py` (2262 строк) на 4 модуля: `admin_views`, `admin_students`, `admin_variants`, `admin_catalog`

### Ученики
- Поддержка π в ответах: принимать без символа, нормализовать `pi`, подсказка в задании
- Кнопка «Повторить ошибки» — создаёт вариант из неправильно решённых заданий
- Уведомления учителя о новых попытках
- Аналитика по номерам заданий (процент выполнения) для ученика и класса

### Проверка
- Массовая проверка выбранных попыток (bulk_grade)
- Оптимизация массовой проверки: один SQL-запрос вместо N+1

### Интерфейс

**Панель учителя (redesign, волна 2):**
- CSS-утилиты: `.table-responsive`, `.form-help`, `.btn-active`, `.row-weak`, `.perf-badge`, `.attempt-meta`, `.section-sep`, `.export-group`, `.card`
- Активный пункт навигации выделен подчёркиванием (`.active` в navbar)
- Stat-карточки на страницах статистики ученика, класса, варианта — иконки и цветовые акценты
- Пустые состояния (empty state с Lucide-иконкой) на всех list-страницах вместо голого текста
- Perf badges вместо inline-цветов в таблицах статистики
- Chart.js: цвет графика из CSS-переменной `--primary` вместо hardcoded hex
- Таблицы: `table-responsive` wrapper везде, hover-подсветка строк
- Кнопки вторичных действий в строках таблиц убраны за `···` dropdown
- Страница детали попытки: CSS-классы `.attempt-meta`, `.status-pending/correct/wrong` вместо inline-стилей
- Формы: `form-help` подсказки на полях, сообщения об успехе без inline-стилей
- Прогресс-бары в аналитике окрашены по результату (зелёный / жёлтый / красный)

**Страницы ученика (redesign, завершён):**
- Страница входа: split-screen 50/50, тёмная левая панель, underline-инпуты, динамический год
- Профиль: зона уровня, инсайт-блоки, тренд результатов, Chart.js с порогом оценки 4
- Выбор варианта: primary-блок активного варианта, лента попыток с цветами оценок, human dates, переключатель «показать все»
- Результаты: персонализированный CTA («что делать дальше») по оценке, human dates для предыдущих попыток
- Иконки Lucide на статусных бейджах, SVG-заглушки вместо эмодзи в пустых состояниях

### CI и деплой
- Автодеплой через GitHub Actions
- Запуск тестов и линтера (ruff) в CI перед деплоем
- Мониторинг ошибок через Sentry (опционально, через `SENTRY_DSN`)
- Автобэкап PostgreSQL через cron
- Обновлены actions/checkout до v4.2.2 и actions/setup-python до v5.6.0
- Проверка миграций через `makemigrations --check --dry-run`
- Вывод хэша коммита в логах деплоя

### Улучшения
- Индекс `(student, variant)` на таблице попыток — ускоряет проверку лимита попыток при выборе варианта
- Варианты «повтор ошибок» (`ошибки_*`) скрыты из списка вариантов и экспорта ZIP — учитель не видит технический мусор

### Багфиксы
- Статистика класса больше не показывает результат >100% (некорректный JOIN в annotate)
- Excel-выгрузка аналитики класса скачивается как файл, а не открывается в Яндексе (UTF-8 filename*)
- Название ZIP-архива: убрано лишнее слово «Варианты», добавлен корректный filename*
- Улучшен текст уведомления о попытках к проверке на дашборде + кнопка «Перейти к проверке»
- Единицы измерения в правильных ответах («0.4 мм» → «0.4»)
- Состояние фоновых задач между воркерами Gunicorn (FileBasedCache вместо LocMemCache)
- Изображения не вставлялись в DOCX
- «General condition (click to expand)» попадало в текст при экспорте в Word
- Название книги автора попадало в контекст при PDF-импорте
- Ответы через `|` (несколько правильных вариантов)
- Общее условие для заданий 1–5 повторялось в DOCX; теперь выводится один раз
- Заглушены email-ошибки от ботов с чужими доменами (DisallowedHost)
- Race condition при старте экзамена — select_for_update блокирует дублирование попыток
- Дубли ФИО в Excel-импорте проверяются до сохранения
- N+1 в статистике класса устранён через annotate
- PDF-импорт: meta-refresh заменён на fetch polling, улучшены состояния ошибок
- Таймер в задании: авто-завершение теперь флашит ответы через событие submit перед финишем
- Единый флоу авто-завершения при истечении таймера (устранено дублирование логики)
- Названия оценок и неразрывные пробелы в фразе с результатом исправлены
