# ТЗ: Платформа пробных вариантов ОГЭ/ЕГЭ по математике

## 1. Общее описание

Веб-приложение на Python для проведения пробных ОГЭ/ЕГЭ по математике. Задания парсятся с сайтов ФИПИ, "Распечатай и реши", Динамика. Админ (учитель) создаёт варианты, ученики решают их онлайн, результаты сохраняются и доступны для анализа.

**Предмет:** математика (позже возможно расширение).

**Форматы экзаменов:**
- ОГЭ — 25 заданий (1 часть), таймер 3ч 55мин
- ЕГЭ профиль — 18 заданий (1 часть), таймер 3ч 55мин
- ЕГЭ база — 21 задание (1 часть), таймер 3ч

> Часть 2 (развёрнутые ответы) — будет добавлена позже.

---

## 2. Роли

| Роль | Доступ |
|------|--------|
| **Ученик** | Вход по ФИО + пароль, решение вариантов своего типа (ОГЭ/ЕГЭ), просмотр своей статистики |
| **Админ (учитель)** | Вход через `/admin` (логин + пароль). Создание/редактирование вариантов, управление учениками и классами, просмотр результатов |

---

## 3. Страницы ученика

### 3.1 Вход
- Поля: ФИО + пароль
- Учётные записи создаёт админ (указывает ФИО, пароль, класс и тип экзамена)
- Админ передаёт пароль ученику
- При неверном вводе — сообщение **"Неверное ФИО или пароль"** (без уточнения что именно неверно)
- Без ограничения попыток
- Кнопки "Забыл пароль" нет — сброс только через админа
- **Один аккаунт = одна активная сессия** (при входе с другого устройства предыдущая сессия завершается)
- После входа — переход к выбору варианта

### 3.2 Выбор варианта
- Ученик видит только варианты своего типа экзамена
- Кнопка **"Случайный вариант"**
- Поле ввода **ID варианта** (например, `092`) + кнопка **"Начать"**
  - Если ID не существует — красным шрифтом под полем: **"Вариант с таким ID не найден"**
- **Список ранее решённых вариантов** с результатами (ниже на странице)

### 3.3 Решение варианта
- Все задания на одной странице (скроллом вниз)
- После каждого задания — поле для ввода ответа
- **Сверху (фиксированная панель):**
  - Таймер (длительность зависит от типа экзамена)
  - Навигация по номерам заданий (кликабельные номера — скролл к заданию)
  - Если время вышло — автоматический переход к итогам
- **Внизу** — кнопка **"Завершить"**
  - При нажатии — модальное окно подтверждения: **"Вы уверены? Не отвечено: N заданий"** (если есть пустые)
  - Если все задания заполнены: **"Вы уверены, что хотите завершить?"**
- **Автосохранение** ответов по ходу решения (если вкладка закроется — ответы не потеряются)
- Ученик может решать один и тот же вариант несколько раз, все попытки сохраняются

### 3.4 Итоги
- Общий результат: первичные баллы → перевод в оценку / тестовые баллы
- **Время решения** (сколько ученик потратил)
- **Сравнение с предыдущими попытками** этого же варианта (если были)
- Кнопка **"Посмотреть неправильные"** — раскрывается список:
  - Номер задания
  - Ответ ученика
  - Правильный ответ
- Кнопка **"Посмотреть весь вариант"** — все задания с правильными и неправильными ответами
- Кнопка **"Решить ещё"** — возврат к экрану выбора варианта

### 3.5 Профиль (кнопка в правом верхнем углу)
- Отображается ФИО
- По клику — переход в личную статистику:
  - **Средний балл** по всем вариантам
  - **График прогресса** (изменение баллов со временем)
  - Список решённых вариантов (все попытки)
  - По каждому варианту — результат по заданиям (зелёный/красный кружок)

## 4. Админ-панель (`/admin`)

### 4.1 Управление вариантами
- Создание нового варианта с присвоением номера (например, `092`)
- Указание типа варианта: ОГЭ / ЕГЭ профиль / ЕГЭ база
- Редактирование существующего варианта
- **Деактивация варианта** (скрыть от учеников, не удаляя)
- **Дублирование варианта** (копия для редактирования)
- Каждое задание содержит:
  - Текст задания (возможно с изображением)
  - Правильный ответ
  - Источник (ФИПИ / Распечатай и реши / Динамика)

### 4.2 Управление классами
- Создание / редактирование / удаление классов (например, "9А", "11Б")
- Каждый класс привязан к типу экзамена (ОГЭ / ЕГЭ профиль / ЕГЭ база)

### 4.3 Управление учениками
- Добавление ученика: ФИО, пароль, класс (тип экзамена наследуется от класса)
- **Массовое добавление** — загрузка Excel-файла (столбцы: ФИО, пароль, класс)
- Редактирование / удаление ученика
- Сброс пароля
- Фильтрация по классу и типу экзамена

### 4.4 Статистика учеников
- Таблица всех учеников (с фильтрацией по классам)
- По клику на ФИО — статистика ученика:
  - Средний балл, график прогресса
  - Список решённых вариантов (все попытки)
  - По каждому варианту — номер + результат по заданиям (🟢 / 🔴)
  - Первичные баллы и оценка по каждой попытке
  - По клику на номер варианта — вариант с ответами ученика

---

## 5. Проверка ответов

- Автоматическая проверка (только 1 часть)
- Эквивалентные ответы считаются правильными (например, `0.5` = `1/2`)
- Допустимые форматы: целые числа, десятичные дроби, обыкновенные дроби, отрицательные числа, последовательности цифр
- Нормализация ответа перед сравнением

## 8. Модель данных (основные сущности)

- **Class** — название (например, "9А"), тип экзамена (ОГЭ / ЕГЭ профиль / ЕГЭ база)
- **Student** — ФИО, пароль (хэш), класс (FK → Class)
- **Admin** — логин, пароль (хэш)
- **Variant** — номер, тип (ОГЭ / ЕГЭ профиль / ЕГЭ база), активен/неактивен, длительность таймера
- **Task** — вариант (FK), номер в варианте, текст, изображение (опционально), правильный ответ, источник
- **Attempt** — ученик (FK), вариант (FK), дата, время начала/конца, первичный балл, оценка
- **Answer** — попытка (FK), задание (FK), ответ ученика, правильность

---

## 9. Система оценивания

- Подсчёт первичных баллов (каждое задание = определённое кол-во баллов)
- Перевод первичных баллов в оценку (ОГЭ) или тестовые баллы (ЕГЭ)
- Таблицы перевода обновляются в соответствии с актуальными шкалами
