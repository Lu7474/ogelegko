# 📐 Пробные ОГЭ/ЕГЭ — платформа для учителей математики

Веб-приложение для проведения пробных экзаменов по математике в школе.  
Учитель создаёт варианты и ведёт учеников — ученики решают онлайн, результаты сохраняются автоматически.

---

## Возможности

### Для учителя (админ-панель)
- Создание вариантов вручную, через **каталог заданий** или импортом с **СдамГИА**
- Управление классами и учениками (в т.ч. массовый импорт из Excel)
- Просмотр результатов по каждому ученику и классу
- Ручная проверка заданий части 2 (развёрнутые ответы)
- Экспорт результатов в `.xlsx`
- Уведомления о новых завершённых попытках в реальном времени

### Для ученика
- Вход по ФИО и паролю
- Решение вариантов своего типа (ОГЭ / ЕГЭ профиль / ЕГЭ база) с таймером
- Автосохранение ответов — при закрытии вкладки прогресс не теряется
- Просмотр результатов: правильные/неправильные ответы, время, оценка
- Личная статистика и история попыток

## Быстрый старт

### 1. Клонирование и окружение

```bash
git clone <repo-url>
cd EGE
python -m venv env
source env/bin/activate        # Windows: env\Scripts\activate
pip install -r requirements.txt
```

### 2. Настройка

```bash
# Применить миграции
python manage.py migrate

# Создать суперпользователя (учитель-админ)
python manage.py createsuperuser
```

### 3. Запуск

```bash
python manage.py runserver
```

Откройте [http://localhost:8000](http://localhost:8000) — страница входа для учеников.  
Админ-панель: [http://localhost:8000/admin/](http://localhost:8000/admin/)

## Каталог заданий

Центральная база заданий, не привязанных к конкретному варианту.

- **Добавление вручную** — через форму
- **Импорт с СдамГИА** — по ссылке на задание или целый вариант (фоново)
- **Неопределённые задания** — задания без номера, которым учитель назначает тип (1–25) одним кликом
- **Создание варианта из каталога** — модальное окно с поиском, фильтрами по источнику, защитой от дублей и индикатором заполненных слотов

## Импорт учеников из Excel

Перейдите в **Ученики → Импорт Excel**.  
Файл должен содержать столбцы: `ФИО`, `Пароль`, `Класс`.  
Класс должен уже существовать в системе.

## Переменные окружения

| Переменная            | По умолчанию          | Описание                        |
|-----------------------|-----------------------|---------------------------------|
| `DJANGO_SECRET_KEY`   | insecure-key          | Секретный ключ Django           |
| `DJANGO_DEBUG`        | `True`                | Режим отладки                   |
| `DATABASE_URL`        | SQLite                | URL подключения к БД            |
| `ALLOWED_HOSTS`       | `*`                   | Разрешённые хосты               |
| `CLOUDINARY_URL`      | —                     | Cloudinary для хранения медиа   |

## Проверка ответов

Ответы нормализуются перед сравнением:

- `0,5` = `0.5` = `1/2`  
- Пробелы и регистр игнорируются  
- Последовательности: `1;2;3` — порядок важен  
- Задания части 2 проверяются учителем вручную

## Деплой

Проект готов к деплою на любой Linux-сервер с Python.

```bash
# Продакшн-сервер
gunicorn config.wsgi:application --bind 0.0.0.0:8000

# Статические файлы раздаёт WhiteNoise — доп. настройка nginx не требуется
python manage.py collectstatic --noinput
```

Для базы данных задайте `DATABASE_URL` в переменных окружения (поддерживается PostgreSQL через `dj-database-url`).
